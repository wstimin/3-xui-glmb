# 十夜管理系统部署教程

本教程适合把十夜 3-xui 用户管理系统部署到 Linux 服务器。当前版本使用 MySQL 作为业务数据存储，推荐使用一键脚本安装，也可以在 1Panel、宝塔等面板中按 Node.js 项目部署。

项目按 3-xui 3.4.1 版本开发和测试。其他版本通常也可以接入，但不同版本或魔改版的 API 路径、字段可能不同，部署后请先在后台测试 3-xui 节点连接和用户同步功能。流量超限续费恢复依赖 3-xui 的 `POST /panel/api/clients/resetTraffic/{email}` 接口，推荐优先使用 API Token 对接。

## 1. 一键安装

在 Linux 服务器上执行：

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/wstimin/3-xuiguanli-shangye/main/install.sh)
```

脚本会自动完成：

- 检查 root、systemd 和包管理器
- 安装基础依赖、Git 和 Node.js 20+
- 未提供外部 MySQL 配置时，自动安装本机 MySQL/MariaDB 兼容服务
- 创建数据库、数据库用户和随机密码
- 拉取项目到 `/opt/shiye-management-system`
- 执行 `npm ci --omit=dev`，失败时回退到 `npm install --omit=dev`
- 执行 `node --check server.js`、`node --check public/app.js` 和 `node --check public/user.js`
- 写入 `/etc/default/shiye-management-system`
- 创建并启动 systemd 服务
- 可选安装 Nginx 反向代理
- 可选通过 Certbot 申请 Let's Encrypt HTTPS 证书

安装完成后默认访问：

```text
用户入口：http://服务器IP:3388/
管理员入口：http://服务器IP:3388/admin
```

默认管理员账号：

```text
账号：admin
密码：admin123
```

公网部署后建议立即进入后台修改默认账号和密码。

## 2. 常用安装模式

只使用 IP + 端口访问，不安装 Nginx：

```bash
ENABLE_NGINX=no \
bash <(curl -fsSL https://raw.githubusercontent.com/wstimin/3-xuiguanli-shangye/main/install.sh)
```

安装 Nginx，使用域名访问，但暂时不申请 HTTPS 证书：

```bash
ENABLE_NGINX=yes \
ENABLE_HTTPS=no \
DOMAIN=panel.example.com \
bash <(curl -fsSL https://raw.githubusercontent.com/wstimin/3-xuiguanli-shangye/main/install.sh)
```

安装 Nginx，并自动申请 HTTPS 证书：

```bash
ENABLE_NGINX=yes \
ENABLE_HTTPS=yes \
DOMAIN=panel.example.com \
CERTBOT_EMAIL=admin@example.com \
bash <(curl -fsSL https://raw.githubusercontent.com/wstimin/3-xuiguanli-shangye/main/install.sh)
```

使用已有 MySQL：

```bash
MYSQL_HOST=127.0.0.1 \
MYSQL_PORT=3306 \
MYSQL_USER=shiye \
MYSQL_PASSWORD='请换成强密码' \
MYSQL_DATABASE=shiye_management \
ADMIN_PATH=/admin \
bash <(curl -fsSL https://raw.githubusercontent.com/wstimin/3-xuiguanli-shangye/main/install.sh)
```

如果本机 MySQL 的 root 账号设置了密码，可以传入：

```bash
MYSQL_ROOT_PASSWORD='你的MySQLRoot密码' \
bash <(curl -fsSL https://raw.githubusercontent.com/wstimin/3-xuiguanli-shangye/main/install.sh)
```

自定义端口、安装目录或服务名：

```bash
PORT=8080 \
APP_DIR=/opt/my-shiye-panel \
APP_NAME=x-uiguanli \
bash <(curl -fsSL https://raw.githubusercontent.com/wstimin/3-xuiguanli-shangye/main/install.sh)
```

## 3. 面板部署

1Panel、宝塔等面板部署方式仍然按 Node.js 项目处理：

1. 在面板中创建 MySQL 数据库和数据库账号。
2. 上传或拉取完整项目代码。
3. 确认运行目录根部能看到 `package.json`、`server.js` 和 `public/`。
4. 在项目目录执行 `npm install --omit=dev`。
5. 启动命令填写 `npm start` 或 `node server.js`。
6. 项目端口填写 `3388`，如果自定义 `PORT`，则填写自定义端口。
7. 在面板网站中把域名反向代理到 `http://127.0.0.1:3388`。
8. 首次访问管理员入口，例如 `https://你的域名/admin`，按页面提示完成安装向导。

如果想使用网页安装向导，不要提前在面板环境变量里填写 `MYSQL_HOST`、`MYSQL_USER`、`MYSQL_PASSWORD`、`MYSQL_DATABASE` 或 `DATABASE_URL`。系统只有在没有数据库环境变量，并且不存在 `data/config.json` 时，才会显示安装向导。

面板部署的详细步骤可以查看：

- [1Panel部署教程.md](./1Panel部署教程.md)
- [宝塔部署教程.md](./宝塔部署教程.md)

## 4. 配置 3-xui 节点

进入后台“3x-ui 节点”，点击“添加 3x-ui 节点”。推荐优先填写 API Token，账号和密码可以留空。

示例一：

```text
3-xui 地址：http://example.com:2053/
协议：http
地址：example.com
端口：2053
基础路径：
API Token：填写 3-xui 的 API Token
```

示例二：

```text
3-xui 地址：https://example.com:2053/custompath/
协议：https
地址：example.com
端口：2053
基础路径：custompath
API Token：填写 3-xui 的 API Token
```

添加后点击“测试”。能显示可用 Inbound ID，说明连接成功。

## 5. 自动停用和续费恢复

系统会定时维护用户节点状态：

- 本地节点到期后会自动停用，并同步停用远端 3-xui client。
- 远端 3-xui client 流量用完或被远端停用时，本地节点会同步停用。
- 因流量超限被停用的节点续费后，系统会调用 3-xui 官方 `POST /panel/api/clients/resetTraffic/{email}` 接口，清空该 client 流量并重新启用。
- 用户自助续费采用严格逻辑：先同步 3-xui 成功，再扣除余额并保存本地续费记录。

如果续费后仍不能连接，请先检查 3-xui API Token 权限、面板版本是否支持 `resetTraffic/{email}`，以及同步日志中的远端错误信息。

## 6. 常用命令

```bash
systemctl status shiye-management-system
systemctl restart shiye-management-system
journalctl -u shiye-management-system -f
```

手动语法检查：

```bash
node --check server.js
node --check public/app.js
node --check public/user.js
```

查看服务环境配置：

```bash
cat /etc/default/shiye-management-system
```

修改配置后重启：

```bash
nano /etc/default/shiye-management-system
systemctl restart shiye-management-system
```

## 7. 更新项目

推荐重新执行一键安装命令：

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/wstimin/3-xuiguanli-shangye/main/install.sh)
```

脚本会覆盖程序文件，保留 `data/` 目录，并重启服务。业务数据保存在 MySQL 中。

如果使用 Git 手动更新：

```bash
cd /opt/shiye-management-system
git pull
npm install --omit=dev
node --check server.js
node --check public/app.js
node --check public/user.js
systemctl restart shiye-management-system
journalctl -u shiye-management-system -n 80 --no-pager
```

## 8. 数据备份

需要备份：

- MySQL 数据库
- `/etc/default/shiye-management-system`
- `data/.secret` 或 `APP_SECRET`
- `data/config.json`，如果你通过网页安装向导保存数据库配置

MySQL 备份示例：

```bash
mysqldump -u shiye -p shiye_management > shiye_management.sql
```

恢复示例：

```bash
mysql -u shiye -p shiye_management < shiye_management.sql
```

如果丢失 `APP_SECRET` 或 `data/.secret`，之前保存的 3-xui Token、密码、SOCKS 密码和支付密钥将无法解密。

## 9. 安全建议

- 公网部署建议使用 HTTPS。
- 公网部署建议修改默认管理员账号和密码。
- 不要公开 `data/.secret`、`data/config.json` 和 `/etc/default/shiye-management-system`。
- 不要公开 3-xui API Token、支付密钥、支付宝私钥、易支付商户密钥和数据库密码。
- 如果不用 Nginx，安全组需要放行 `3388`。
- 如果使用 Nginx，建议只放行 `80/443`，不要公开 `3388`。
